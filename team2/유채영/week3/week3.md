# 색션 4 : 웹 개발의 역사와 API 기초

## 웹 개발의 역사와 패러다임의 변화
웹은 단순한 정보 공유를 넘어, 사용자 중심의 복합적인 서비스로 진화해 왔음.

### 웹의 성장과 분업화
* **기존 (초창기 웹):** 
  * 사용자의 목적이 단순 **정보 조회**였기에 한 사람이 페이지의 모든 요소를 관리 가능했음.
  
* **현대적 웹의 등장:** 
  * 단순 조회를 넘어 **CRUD(생성, 조회, 수정, 삭제)**, 로그인, 유저 맞춤형 데이터 제공 등 기능이 고도화됨.
  *  동일한 디자인 틀 안에서 데이터만 동적으로 변하는 구조가 필요해짐.
  
* **효율적인 개발 체계:** 기존 방식의 비효율을 해결하기 위해 화면과 로직을 분리하여 전문적으로 개발하기 시작함.
  * **프론트엔드(Frontend):** 사용자 화면 개발 전담.
  * **백엔드(Backend):** 로직 설계, 데이터 연산 및 서버 관리 전담.

---

## 백엔드와 API의 이해
백엔드 개발의 핵심은 외부와 소통하는 접점을 만들고 기능을 구현하는 것임.

### API (Application Programming Interface)
* **정의:** 'Interface(서로 마주하다)'라는 의미대로, 서로 다른 소프트웨어가 소통하기 위한 **접점(규격)**을 의미함.
* **핵심 가치:** 내부 데이터를 직접 노출하지 않아 **보안**을 유지함.
  * 사용자가 내부의 복잡한 동작 방식을 몰라도 원하는 기능을 수행할 수 있는 **채널**을 제공함.

* **벡엔드 개발자의 역할:** 단순 인터페이스 제공을 넘어, 그 안에서 돌아가는 **내부 로직과 데이터를 설계하고 생성**하는 것.

### REST API
* **정의:** 인터넷 상의 약속인 **HTTP 프로토콜**을 철저히 준수하여 설계된 API.
* **특징:** 각 자원(Resource)을 이름으로 구분하여 상태를 주고받는 방식.
  * **RESTful API:** HTTP의 설계 원칙을 매우 엄격하게 지키며 구성된 API 지칭.


# 색션 5 : 개발 환경 및 스프링 부트

## 개발 툴 : IntelliJ
* **Community Edition:** 무료 버전 사용.
* **학습 환경:** 스프링 부트를 사용하면 무료 버전으로도 웹 개발이 충분히 가능함.
  
---

## 스프링 부트의 강점 (웹 서버)
* **스프링:** 웹 서버를 직접 설치하고 설정해야 하는 번거로움이 있음.
* **스프링 부트:** 웹 서버가 내부에 **내장**되어 있음.
  * 서버 설치 없이 바로 실행 가능.
  * 설정이 간편하여 입문자가 시작하기에 매우 효율적임.
